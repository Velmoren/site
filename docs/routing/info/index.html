Права доступа и роутинг

Проверка прав пользователей осуществляется в папке router.

router.js

Основной файл регламентирующий роутинг во Vue.JS.
Пример роута /login

{
    path: '/login',
    name: 'login',
    component: LoginView,
    meta: {
        middleware: [
            guest
        ]
    }
}

path - путь после домена
name - метка именованного маршрута. По ней можно передать данные для определенных компонет
component - компонент представления
meta - набор middlewares для проверки доступа

Последовательность логики

Когда пользователь заходит на сайт первым делом отрабатывает метод router.beforeEach который вызывает глобальные навигационные хуки

  1. улетает запрос на сервер с проверкой доступа для данного user:
  • если проверка пройдена - редиректит на /sales/table
  • если нет - name для user присваивается guest. При каждой попытке зайти на сайт guest будет перенаправлен на /login

Если user.name не равен guest и не пустой идет перенаправление на /sales/table

Далее каждый url будет проходить проверки на auth и допуски к разным участком кода. За данные проверки отвечают middleware

Middleware

Набор различных middleware. Название файлов указывается в свойстве middleware. При запуске данный метод принимает
3 параметра: next, store, nextMiddleware

  • next - объект перенаправленая. Указывается name того роута, на который будет произведено перенаправления, если проверка не пройдена
  • store - локальное хранилище данных
  • nextMiddleware - метод, производящий все проверки и производящий логику на их основании
